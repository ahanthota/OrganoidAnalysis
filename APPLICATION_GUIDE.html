ğŸ“‹ Overview

The Organoid Analysis Application is a comprehensive web-based tool designed to analyze organoid cell growth over time using advanced computer vision and deep learning techniques. The application processes time-series images of organoids (typically Day 1, Day 2, Day 3, Day 4, etc.) and provides detailed metrics including organoid count, size distribution, growth rates, and visual analysis.

The application runs as a standalone Windows executable that bundles a local web server, eliminating the need for users to install Python, Flask, or any dependencies. Simply double-click the executable, and a web browser automatically opens to the analysis interface.

ğŸ”¬ Analysis Methods

The application provides 8 different analysis methods, each optimized for different scenarios and image characteristics:

1. Basic Thresholding (Otsu)

How it works: Uses Otsu's automatic thresholding algorithm to separate organoids from background based on pixel intensity values.

Algorithm:

  • Converts image to grayscale
  • Applies Gaussian blur for noise reduction
  • Uses Otsu's method to find optimal threshold
  • Performs automatic background/foreground inversion detection
  • Identifies contours and calculates areas

Best for: Clear, high-contrast images with well-separated organoids

Limitations: Merges touching or overlapping organoids

2. Watershed Segmentation

How it works: Advanced algorithm that can separate touching organoids by treating the image as a topographic map and flooding from local minima (organoid centers).

Algorithm:

  • Preprocessing: Otsu thresholding and morphological opening
  • Distance transform to find organoid centers
  • Marker-based watershed segmentation
  • Separates touching objects based on intensity peaks

Best for: Images with touching or clumped organoids

Advantages: Better separation of overlapping objects compared to basic thresholding

3. Hough Circle Transform

How it works: Mathematical method that detects circular shapes in images, useful for spherical organoids. Estimates 3D volume from 2D projections.

Algorithm:

  • Edge detection using Canny algorithm
  • Hough circle transform to detect circular shapes
  • Volume estimation: V = (4/3)Ï€rÂ³
  • Can detect overlapping circles

Best for: Spherical organoids, volume estimation, heavily overlapping structures

Unique feature: Provides 3D volume estimates from 2D images

4. StarDist (Deep Learning)

How it works: State-of-the-art deep learning model specifically designed for star-convex object detection. Uses neural networks to learn optimal segmentation boundaries.

Algorithm:

  • Pre-trained convolutional neural network
  • Star-convex polygon representation
  • Handles irregular shapes better than geometric methods
  • Fallback to geometric method if TensorFlow unavailable

Best for: Irregularly shaped organoids, complex morphologies

Requirements: TensorFlow (optional - has fallback)

5. U-Net (Deep Learning)

How it works: Encoder-decoder convolutional neural network architecture originally designed for biomedical image segmentation. Uses skip connections to preserve fine details.

Algorithm:

  • Encoder path: Feature extraction through convolutions
  • Decoder path: Upsampling and feature reconstruction
  • Skip connections: Preserve spatial information
  • Output: Probability map for each pixel
  • Thresholding and post-processing to create binary masks

Best for: Precise boundary detection, complex backgrounds, variable organoid sizes

Unique feature: Provides probability heatmaps showing segmentation confidence

Requirements: TensorFlow (optional - has fallback)

6. Advanced Morphology & Cell Count

How it works: Combines watershed segmentation with advanced morphological feature extraction to estimate cell counts and characterize organoid shape.

Features calculated:

  • Solidity: Ratio of organoid area to convex hull area (measures compactness)
  • Eccentricity: Measure of how elongated an organoid is (0 = circle, 1 = line)
  • Cell Count Estimation: Uses OSCAR-inspired regression: cells = area / average_cell_area

Best for: Detailed morphological analysis, cell proliferation studies

Inspiration: MOrgAna and OSCAR tools

7. ZEISS arivis Pro (Simulation)

How it works: Simulates the 3D volumetric segmentation workflow of commercial microscopy software. Creates wireframe representations and volume estimates.

Algorithm:

  • Multi-scale processing for 3D simulation
  • Volume estimation from 2D projections
  • Wireframe visualization

Best for: Users familiar with commercial tools, 3D analysis workflows

8. AssayScope (Simulation)

How it works: High-throughput screening approach focusing on population-level statistics and homogeneity metrics.

Features:

  • Mean radius calculation
  • Coefficient of Variation (CV)
  • Homogeneity score: 100 - CV%
  • Outlier detection

Best for: High-throughput experiments, population studies, quality control

ğŸ“Š Complete Workflow

Step-by-Step Processing Pipeline

  1. Image Upload
    • User selects multiple images (typically Day 1, Day 2, Day 3, Day 4)
    • Files are saved to static/uploads/ directory
    • Images are sorted by filename to determine temporal order
  2. Image Preprocessing
    • Images are loaded using OpenCV
    • Conversion to grayscale (if needed)
    • Noise reduction using Gaussian blur
    • Contrast normalization
  3. Segmentation
    • Selected analysis method is applied to each image
    • Organoids are separated from background
    • Individual organoids are identified and labeled
  4. Feature Extraction
    • Area calculation (pixelsÂ²)
    • Contour detection for boundaries
    • Additional features based on selected method (circularity, volume, etc.)
    • Filtering: Small objects (< 80-100 pixels) removed as noise
  5. Statistical Analysis
    • Count: Number of detected organoids
    • Average size: Mean area across all organoids
    • Total area: Sum of all organoid areas
    • Growth rate: Percentage change from previous day
  6. Visualization Generation
    • Debug images created showing detected boundaries
    • Contours overlaid on original images
    • Method-specific visualizations (heatmaps, wireframes, etc.)
    • Saved to static/uploads/debug_output_*/
  7. Results Presentation
    • Statistics displayed in cards
    • Growth trends shown in interactive charts
    • Visual gallery of processed images
    • Export capabilities through browser

ğŸ“– Usage Guide

Getting Started

  1. Download and Extract
    • Download OrganoidAnalysis.exe from GitHub Actions artifacts
    • No installation required - just run the executable
  2. Launch Application
    • Double-click OrganoidAnalysis.exe
    • Wait 5-10 seconds for the server to start
    • Browser will open automatically to http://127.0.0.1:5174
  3. Upload Images
    • Click "Click to upload" button
    • Select multiple images (Day 1, Day 2, etc.)
    • Supported formats: JPG, PNG, JPEG
    • Images are automatically sorted by filename
  4. Select Analysis Method
    • Choose from dropdown menu
    • Read method descriptions to select the best one
    • Recommended: Watershed for clumped organoids, U-Net for precision
  5. Run Analysis
    • Click "Analyze Data" button
    • Wait for processing (typically 10-30 seconds)
    • Results appear automatically when complete
Important Notes:
  • First run may take longer (extracting bundled files)
  • Firewall may prompt for permission - allow localhost connections
  • Keep the browser window open while analyzing
  • Do not close the terminal/console window (if visible)

ğŸ“ˆ Understanding Results

Statistics Cards

The application displays several key metrics:

  • Latest Count: Number of organoids detected on the most recent day
  • Total Growth: Percentage change in organoid count from Day 1 to latest day
  • Average Size: Mean area of organoids in pixelsÂ²
  • Total Area: Sum of all organoid areas (biomass indicator)
  • Growth Rate: Day-to-day percentage changes

Charts and Visualizations

  • Growth Trend (Count): Line chart showing organoid count over time
    • Increasing: Suggests proliferation
    • Decreasing + Size increasing: Suggests fusion
    • Stable: Mature culture
  • Size Distribution: Bar chart showing average size or total area
    • Upward trend: Confirms growth
    • Plateau: Growth stabilization

Visual Analysis

Each processed image shows:

  • Original image with detected boundaries overlaid
  • Color-coded contours indicating organoids
  • Method-specific visualizations:
    • Watershed: Red contours with yellow boundaries
    • U-Net: Green contours with probability heatmap
    • Hough: Magenta circles showing detected spheres

Port and Networking

  • Port: 5174 (configurable via PORT environment variable)
  • Host: 127.0.0.1 (localhost only - no external access)
  • Protocol: HTTP (not HTTPS - local only)
  • Security: Completely local - no data leaves the machine

Performance Characteristics

  • Startup time: 5-15 seconds (first run may be longer)
  • Analysis time: 10-30 seconds per image set (depends on method and image size)
  • Memory usage: ~200-500 MB (depends on image sizes)
  • Disk space: ~100-200 MB for the executable

System Requirements

  • OS: Windows 10 or 11 (64-bit)
  • RAM: 4 GB minimum, 8 GB recommended
  • Disk: 500 MB free space
  • No Python required - Everything is bundled
  • No internet required - Fully offline operation

Dependencies Included

  • Python 3.9.13 runtime
  • Flask 2.0+
  • OpenCV (cv2) 4.5+
  • NumPy 1.21+
  • Werkzeug, Jinja2 (Flask dependencies)
  • TensorFlow (optional, for U-Net/StarDist)
  • Chart.js (for frontend visualizations)

Fallback Mechanisms

Smart Error Handling: The application includes fallback mechanisms:

  • If TensorFlow is unavailable, U-Net and StarDist automatically use advanced image processing fallbacks
  • If a specific method fails, the application gracefully handles errors and reports them to the user
  • All methods have been tested to work without deep learning libraries if needed

ğŸ› ï¸ Troubleshooting

Common Issues

Application won't start

  • Check Windows Defender/Antivirus - may be blocking executable
  • Ensure port 5174 is not in use by another application
  • Try running as administrator
  • Check Windows Event Viewer for error messages

Browser doesn't open automatically

  • Manually navigate to http://127.0.0.1:5174
  • Check if firewall is blocking the connection
  • Wait 10-15 seconds for server to fully start

Analysis fails or gives incorrect results

  • Check image quality - ensure good contrast between organoids and background
  • Try a different analysis method
  • Verify images are in correct format (JPG, PNG)
  • Check debug output images to see what was detected

Slow performance

  • Large images take longer to process - consider resizing
  • U-Net and StarDist are slower than geometric methods
  • Close other applications to free up memory

Customization

The application can be customized by modifying the source code:

  • Port change: Edit desktop_app.py, line with port = 5174
  • Threshold values: Modify area thresholds in analysis modules
  • Method parameters: Adjust algorithm-specific settings in respective files

Extending Functionality

To add new analysis methods:

  1. Create new file: organoid_analysis_yourmethod.py
  2. Implement function: analyze_organoids_yourmethod(image_paths)
  3. Add to desktop_app.py imports and route handler
  4. Add option to frontend dropdown in templates/index.html
  5. Rebuild with PyInstaller

Batch Processing

For processing multiple image sets:

  • Process one set at a time through the web interface
  • Save results before processing next set
  • For automation, consider using the Python modules directly
